# scientific-journal-pipeline

## Setup

Run these commands from the project folder:

```powershell
cd D:\DE\SWP\scientific-journal-pipeline
```

Activate the virtual environment:

```powershell
.\venv\Scripts\Activate.ps1
```

If PowerShell blocks the activation script, run commands through the venv Python directly:

```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
.\venv\Scripts\python.exe -m src.main
```

Install dependencies:

```powershell
pip install -r requirements.txt
```

## Environment

The project reads SQL Server and OpenAlex settings from `.env`.

Required values:

```env
SQLSERVER_DRIVER=ODBC Driver 17 for SQL Server
SQLSERVER_SERVER=LAPTOP-8HG0UFTL\SQLEXPRESS
SQLSERVER_DATABASE=scientific_journal_tracking_db
SQLSERVER_USERNAME=sa
SQLSERVER_PASSWORD=your_password
SQLSERVER_TRUSTED_CONNECTION=no
SQLSERVER_ENCRYPT=no
SQLSERVER_TRUST_SERVER_CERTIFICATE=yes

OPENALEX_BASE_URL=https://api.openalex.org
```

## Run Pipeline

After activating `venv`, run:

```powershell
python -m src.main
```

If `venv` is not activated, run:

```powershell
.\venv\Scripts\python.exe -m src.main
```

## Reprocess Raw Data

Use this only when existing raw records need to be processed again, for example after changing transform or loader logic.

After activating `venv`, run:

```powershell
python -m src.jobs.reprocess_raw_papers
```

If `venv` is not activated, run:

```powershell
.\venv\Scripts\python.exe -m src.jobs.reprocess_raw_papers
```

This job changes existing `processed` or `failed` raw records back to `pending`, then processes them again into `core.*` tables.

## Notes

- The pipeline now uses SQL Server, not MongoDB.
- Raw OpenAlex responses are stored in `raw.works`.
- Processed canonical data is stored in `core.*` tables.
- External source ids are stored in `*_source_mappings.source_record_id`.
- SQL internal ids such as `paper_id`, `author_id`, `journal_id`, and `keyword_id` are GUID values generated by SQL Server.
